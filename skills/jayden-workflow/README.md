# Jayden Workflow

Personal OpenCode/GSD-lite workflow layer. These skills branch from Matt-style workflows without editing Matt skill files.

- **[caveman-commit](./caveman-commit/SKILL.md)** — Ultra-compressed conventional commit message generator.
- **[caveman-compress](./caveman-compress/SKILL.md)** — Compress natural language memory files into caveman format.
- **[caveman-help](./caveman-help/SKILL.md)** — Quick reference for caveman modes, skills, and commands.
- **[caveman-review](./caveman-review/SKILL.md)** — Ultra-compressed code review comments.
- **[compress](./compress/SKILL.md)** — Legacy alias for caveman memory-file compression.
- **[jayden-workflow](./jayden-workflow/SKILL.md)** — Top-level layered workflow system for Map/New/Pivot -> Plan -> Check -> Execute -> Verify.
- **[gsd-lite-check](./gsd-lite-check/SKILL.md)** — Pre-execution gate for plan coverage, wiring, verification, and decision compliance.
- **[gsd-lite-context](./gsd-lite-context/SKILL.md)** — Defines project-local `.planning/` as the central GSD-lite context spine.
- **[gsd-lite-debug](./gsd-lite-debug/SKILL.md)** — Persistent GSD-lite debugging wrapper around Matt's `diagnose` loop.
- **[gsd-lite-execute](./gsd-lite-execute/SKILL.md)** — Execute approved plans from `.planning/current/*` while following map docs and stop gates.
- **[gsd-lite-new-project](./gsd-lite-new-project/SKILL.md)** — Initialize a lean project-local `.planning/` spine for a new repo or product idea.
- **[gsd-lite-pivot](./gsd-lite-pivot/SKILL.md)** — Reframe a stale or dead project into a new product direction with explicit keep/delete boundaries.
- **[gsd-lite-plan](./gsd-lite-plan/SKILL.md)** — Create decision-complete executable plans using meta-prompting, maps, and Matt architecture language.
- **[gsd-lite-review](./gsd-lite-review/SKILL.md)** — Compact GSD-style code review for changed files.
- **[gsd-lite-verify](./gsd-lite-verify/SKILL.md)** — Goal-backward verification for completed work.
- **[map-codebase-architecture](./map-codebase-architecture/SKILL.md)** — Map a codebase into GSD-compatible project memory docs using GSD mapper structure and Matt-style architecture vocabulary.
